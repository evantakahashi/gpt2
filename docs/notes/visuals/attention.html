Attention — interactive walkthrough

Small example: T = 4 tokens, D = 8 input dim, d_head = 4. One attention head. Edit input tokens or projection weights below — all downstream matrices recompute live.

The whole thing in one picture

   input x       (T, D)
      │
      │ three independent learned linear projections
      ▼
   ┌────┐  ┌────┐  ┌────┐
   │ Q  │  │ K  │  │ V  │     each shape (T, d_head)
   └─┬──┘  └─┬──┘  └─┬──┘
     │       │       │
     │       │       │   step ①    scores = Q · K^T            (T, T)
     └───┬───┘       │   step ②    scores = scores / √d_head
         │           │   step ③    scores += causal mask       (upper-triangle = −∞)
         ▼           │   step ④    weights = softmax(scores)   (rows sum to 1)
      ┌─────┐        │
      │  A  │        │
      └─┬───┘        │
        │            │
        └────┬───────┘   step ⑤    output  = weights · V       (T, d_head)
             ▼
          ┌──────┐
          │ out  │       step ⑥    output projection (linear) back to (T, D)
          └──────┘

Step 0. The input — 4 tokens, each a D=8 vector

Imagine these are token embeddings after the embedding lookup + positional embedding addition. Each row is one token's D=8 vector. You can edit these.

Realistic D = 768. We use D=8 here so you can see every number.

Step 1. Project x into Q, K, V

Three learned linear maps: W_Q, W_K, W_V, each shape (D, d_head). We apply them to every token:

Q = x @ W_Q K = x @ W_K V = x @ W_V

Output shapes: each of Q, K, V is (T, d_head) = (4, 4).

W_Q (D=8, d_head=4)
W_K (D=8, d_head=4)
W_V (D=8, d_head=4)
Q = x @ W_Q (T=4, d_head=4)
K = x @ W_K (T=4, d_head=4)
V = x @ W_V (T=4, d_head=4)
Each row of Q, K, V corresponds to one token. Token i's query is Q[i], its key is K[i], its value is V[i]. Q, K, V are NOT three copies of x — they're three different learned views of x.

Step 2. Attention scores: Q · KT

We compute, for every pair of positions (i, j), how relevant token j is to token i:

scores[i, j] = Q[i] · K[j] (a single number, the dot product)

That's T × T dot products, computed efficiently as one matmul:

scores = Q @ KT shape (T, T)

Higher score = token i's query is more aligned with token j's key. The model has learned W_Q, W_K such that "tokens we should attend to" produce high inner products.

Step 3. Scale by √dhead

The raw scores grow in magnitude with d_head (because we sum d_head products in each dot product). If they're too big, softmax saturates and gradients vanish. Fix:

scaled = scores / √d_head (d_head = 4 here, so divide by 2)
Why exactly √d_head?

If Q[i] and K[j] have entries with mean 0 and variance 1, their dot product has variance d_head (sum of d_head independent products of unit-variance things). To bring variance back to 1, divide by √d_head. Then softmax sees values whose scale doesn't depend on d_head — soft, not saturated.

scaled = scores / √d_head (T, T)

Step 4. Causal mask — token i can only see tokens ≤ i

GPT is autoregressive: position i must predict using only positions 0..i. So we forbid attention to "future" positions by adding −∞ to those score entries:

scaled[i, j] = −∞ whenever j > i

After softmax those become 0 attention weight.

causal mask (1 = keep, 0 = forbid)
Lower triangular ones. Row i = position i; column j = what position i is allowed to attend to.
masked = scaled + (mask==0 ? −∞ : 0)

Step 5. Softmax — turn scores into a probability distribution

For each row of masked, take softmax:

weights[i, j] = exp(masked[i, j]) / Σk exp(masked[i, k])

Each row now sums to 1. The −∞ entries become exactly 0. Higher score → higher weight.

weights = softmax(masked, dim=-1) (T, T)

Row sums (sanity)

Every row should sum to 1.000 — that's the softmax invariant.

Step 6. Apply weights to V — the weighted sum (this is the key insight)

This is the part you were missing in your mental model. The output for token i is NOT a dot product with V — it's a weighted sum of all V rows, where weights come from step 5:

output[i] = Σj weights[i, j] · V[j] (a d_head-dim vector)

All T outputs at once:

output = weights @ V shape (T, d_head)

Why a weighted sum and not a dot product?

We want each output position to be a combination of the V vectors from all relevant input positions. A dot product would produce a scalar; we need a d_head-dim vector to pass to the next layer. The weighted sum mixes information from multiple V vectors in proportion to how relevant each was — that's the whole point of attention.

Step 7. Output projection back to D

In a multi-head setting we'd concatenate all heads' outputs back to D dim before this step. With one head, output is already (T, d_head=4) but the rest of the model expects (T, D=8). So we project once more:

final = output @ W_o W_o shape (d_head, D) final shape (T, D)

This is the c_proj in our codebase. We skip showing it here — it's just another linear and adds nothing conceptually new.

What this is missing (you'll need it when implementing)

Multi-head attention

Real GPT-2 uses n_head = 12. Instead of one big (T, T) attention with d_head=D, it splits D into 12 heads of d_head=64 each, runs attention in parallel for each, and concatenates the outputs back to D. In code: reshape Q, K, V into (B, n_head, T, d_head); matmul becomes (B, n_head, T, T); output is (B, n_head, T, d_head) → reshape to (B, T, D).

Batched inputs

We did one example here. In code there's an outer batch dim B; everything becomes 4-D tensors (B, n_head, T, *).

The combined QKV projection (GPT-2 fuses these)

GPT-2's implementation uses one big nn.Linear(D, 3*D) instead of three (D, D) linears. The output is then split into Q, K, V along the last dim. Same math, one matmul instead of three — faster.

What you'll see in attention.py

The forward method you'll write does all of steps 1–7 in vectorized form, no Python loops. The math is exactly what you saw above; the difficulty is keeping the tensor shapes straight as you reshape for multi-head and back.
