Tensor shape (B=2, T=3, D=6). Each row in the grid is one token's features. LayerNorm normalizes across the last dim (the row) — each row independently.

Input tensor x

γ (weight) and β (bias)

Per-row pipeline

1. compute μ (mean over last dim)

μ_row = mean(x_row)

2. compute σ² (variance over last dim)

σ²_row = mean( (x_row − μ_row)² )

3. normalize

x̂_row = (x_row − μ_row) / √(σ²_row + ε)
After this step every row has mean ≈ 0 and variance ≈ 1.

4. scale and shift

y_row = γ ⊙ x̂_row + β
γ and β are learned, shared across all tokens. They let the network "un-normalize on purpose" per-feature.

LayerNorm vs BatchNorm — which axis?

Same tensor (B=2, T=3, D=6), two different reductions.

LayerNorm normalizes across the feature axis (the row)

Each (b, t) gets its own μ and σ². No coupling across the batch.

BatchNorm normalizes across the batch axis (the column)

Each feature dim gets one μ and σ² computed over all tokens in the batch. Needs cross-rank sync under DDP; breaks at batch=1; train/eval mismatch. Reasons we use LN.

Why does step 3 always give mean=0 var=1?
Subtracting the mean centers the data at 0 by definition. Dividing by √σ² makes the variance equal to 1 (any random variable Z with var V, when divided by √V, has var 1). Adding ε inside the sqrt only matters when σ² is so close to 0 that 1/√σ² would blow up.
